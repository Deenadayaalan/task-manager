# TaskFlow — Project Management App

A task management application with Kanban board, dashboard analytics, and backlog views.

This repo contains **two versions**: the original legacy app and a modernised version generated by Kiro's spec-driven AI pipeline.

## Modernised App (React + Node.js)

| Layer    | Technologies                                          |
|----------|-------------------------------------------------------|
| Frontend | React 18, Vite, React Router v6, @dnd-kit             |
| Backend  | Node.js, Express, better-sqlite3                      |
| Deploy   | Docker (single multi-stage container)                  |

### Features

- Dashboard with stat cards (total, to-do, in progress, done, overdue)
- Kanban board with drag-and-drop between columns
- Backlog table with status/priority/assignee filters
- Full-text search across Board and Backlog views
- Toast notifications on every action (create, update, delete, errors)
- Bulk operations: select multiple tasks, mark done/in-progress, bulk delete
- Task create/edit form with validation
- Task detail view with inline status change
- Responsive sidebar (collapses to icons on mobile)
- 80+ tests (frontend component tests + backend integration tests)

### Quick Start

```bash
# Backend
cd new-backend && npm install && npm run dev

# Frontend (separate terminal)
cd new-frontend && npm install && npm run dev
```

Frontend: http://localhost:5173 (Vite proxies /api to backend)
Backend API: http://localhost:3001/api/tasks

### Run Tests

```bash
cd new-backend && npm test     # 17 integration tests
cd new-frontend && npm test    # 63+ component tests
```

### Docker (Production)

```bash
# Single container with built frontend served by Express
docker build -t taskflow .
docker run -p 3001:3001 taskflow
```

### Docker Compose (Development)

```bash
docker compose up --build
# Frontend: http://localhost:5173
# Backend:  http://localhost:3001
```

### Project Structure

```

├── src/

│   │                          # TaskCard, ToastContainer, Layout
│   ├── contexts/              # NotificationContext (toast system)
│   ├── services/              # api.js (fetch wrapper)
│   └── index.css              # All styles (responsive, dark sidebar, badges)
new-backend/                   # Node.js + Express API
├── server.js                  # Express app, CORS, static serving, seed
├── db.js                      # SQLite setup (better-sqlite3)
├── routes/tasks.js            # REST endpoints + bulk operations
├── services/taskService.js    # Business logic
├── models/task.js             # Data access layer
├── seed.js                    # 10 seed tasks matching original
└── tests/                     # Integration tests (vitest + supertest)
```

### API Endpoints

```
GET    /api/tasks              List all (?status, ?priority, ?assignee, ?search)
GET    /api/tasks/:id          Get by ID
POST   /api/tasks              Create
PUT    /api/tasks/:id          Update
PATCH  /api/tasks/:id/status   Update status only
DELETE /api/tasks/:id          Delete
POST   /api/tasks/bulk-status  Bulk update status { taskIds, status }
POST   /api/tasks/bulk-delete  Bulk delete { taskIds }
```

## Spec-Driven Development

The modernisation was driven by a Kiro spec at `.kiro/specs/taskflow-modernisation/`:

- `requirements.md` — Functional and non-functional requirements
- `design.md` — Architecture, component mapping, API contract
- `tasks.md` — 15 implementation tasks (all completed)

The spec preserved 100% of the original API contract and business rules while adding search, notifications, and bulk operations.
